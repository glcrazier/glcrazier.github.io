JavaScript First Acquaintance

2016-01-27

After years of programming with C++ and Java, I’m a little tired of writing lots of code. To implement a simple function such as initializing an array or a map, it takes a lot of lines of code. I hate this. So I thought if there is some language which could relieve from such hell. Then JavaScript comes to me.

Data

As a loose type language, JavaScript never checks types of variables and even do conversion to match desired type. JavaScript has two types of objects:

Primitive

It includes basic data types like Numbers, Strings and Booleans. They are mutable that any changes on their properties would not take any effect.

Non-Primitive

Unlike primitive data, we can manipulate non-primitive objects by enumerating, adding, updating and deleting their properties. And properties can be configured as read-only in ECMAScript5.

Variable

Declaration

A variable can be declared via var keyword.

Not like those strict type language, programmers don’t need to explicit assign type to a variable. You can also declare a variable without var which means it is declared in global scope.

Null vs. undefined

undefined is used to indicate variable that is not declared. Because JavaScript is an interpretor programming language that it’s impossible to check if a variable is declared ahead of execution, that’s why we need to adopt such a keyword.
null is like other languages, used to indicated variable that is not initialized.

Scope

What’s special about scope in JavaScript is that it never works as you think in Java language. In Java, we are able to construct a scope by creating a pair of curly braces which is not the case in JavaScript. When a variable is declared, it is associated with a scope which could be a global one or a local one of a function. When encoutering a variable, the interpretor searches for its declaration in current scope where the code is executed. If no matching is found, it will search in current scope’s parent scope until the top scope which is global scope. We call all scopes involved as a scope chain.
