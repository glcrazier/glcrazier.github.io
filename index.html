JavaScript First Glance

2016-01-27

After years of programming with C++ and Java, I’m a little tired of writing lots of code. To implement a simple function such as initializing an array or a map, it takes a lot of lines of code. I hate this. So I thought if there is some language which could relieve from such hell. Then JavaScript comes to me.

Good

===

Type Free

As a newcomer of JavaScript, you will get that it’s a language full of freedom. It rarely checks variable type, so you can forget those keywords like int, float and double etc. All you need is to use a var keyword to declare a variable. Sometimes you don’t even have to use var. The difference between these two ways of declaring variables will be discussed later.

Easy Run

The next suprising thing is that unlike Java or C++, you don’t need to set up any development environment to build you code and run the executable. The web browser interpretes and run JavaScript code at runtime. It’s relatively up to you to organize your JavaScript code. You can put all your code into a single html file. You can also separate your code into a couple of files for better modulized and future reusability.

Copy the following snippet into a new file, save it with suffix html. Open it with your web browser and see what’s going on.

Review of The Art of Readable Code

2016-01-27

The most benefit of writing readable code is to help others read your code and you share your thoughts inside the code. After long period without touching the code, we can also pick up those “old granny” easily.

Name and Method

  1. It must be clear and not confusing. Try re-reading names to see if it could cause misunderstanding.
  2. Try not using names like tmp, i, j. Every variable should have some meaning.
  3. Don’t use project specific abbreviations.
  4. To name a boolean, use words like is or has to indicate it’s a boolean variable.

Code Looking

  1. If multiple blocks of code are doing familiar things, try to give them the same silhouette.
  2. Aligning parts of the code into ‘columns’ can make code easy to skim through.
  3. If code mentions A, B, and C in one place, don’t say B, C, and A in another. Pick a meaningful order and stick with it.
  4. Use empty lines to break apart large blocks into logical “paragraphs”.

Comment

  1. If what a given method does can be understood by its name, don’t use comment.
  2. Comment is used to show your insight about the code.

About organization forms

Organizations of successful companies may differ in many forms. Some company have no titles and everyone works in a very fair environment. Some companies may allow their employees to spare soem amount of their time to work on other stuff like Google. Some companies may be like Apple that everything is counted on Steve Jobs. LEGO is a company with levels and titles. And he requires all members to focus on company’s work. There doesn’t exist the best organization form.
